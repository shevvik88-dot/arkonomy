Cybersecurity Policy

Last updated: April 20, 2026

Arkonomy takes the security of your financial data seriously. This policy describes the technical and organisational measures we use to protect your account and personal information.

1. Infrastructure Security

Arkonomy's backend is built on Supabase, hosted on AWS infrastructure. All data is encrypted in transit using TLS 1.2 or higher and encrypted at rest using AES-256.

Row-level security (RLS) policies are enforced at the database layer, ensuring that each authenticated user can only read and write their own records — even in the event of an application-layer misconfiguration.

2. Authentication

User accounts are secured through Supabase Auth, which provides:

  • Secure password hashing using bcrypt.
  • Short-lived JWT access tokens with automatic refresh.
  • Email-based verification for new accounts.
  • Protection against brute-force login attempts via rate limiting.

We strongly recommend using a unique, strong password for your Arkonomy account and enabling any available two-factor authentication options.

3. Bank Connection Security

Bank connections are handled exclusively through Plaid, a regulated financial data aggregator. Arkonomy never sees, stores, or transmits your bank username, password, or MFA codes. All credential exchange occurs directly between you and Plaid within their secure, isolated environment.

Arkonomy receives only read-only access tokens and transaction data — it cannot initiate transfers or modify your bank account in any way.

4. API & Edge Function Security

All server-side logic runs as Supabase Edge Functions behind authentication middleware. API keys and secrets are stored as environment variables and are never exposed to the client. Requests to third-party services (Plaid, Alpaca, Anthropic) are made exclusively from server-side functions.

5. Data Minimisation

We collect only the data required to provide the service. Raw transaction data is used locally to generate spending summaries; only anonymised, aggregated figures are forwarded to Anthropic's API for AI insight generation. No personally identifiable information is included in AI prompts.

6. Third-Party Security

We rely on the following third-party providers, each of which maintains their own security certifications and programmes:

Supabase

SOC 2 Type II compliant. Provides database, authentication, and serverless compute. supabase.com/security

Plaid

PCI DSS Level 1 and SOC 2 Type II certified. Handles all bank credential exchange and financial data aggregation. plaid.com/legal/privacy-notice

Alpaca Markets

FINRA-registered broker-dealer subject to SEC and FINRA security requirements. Brokerage accounts are held with Alpaca, not Arkonomy. alpaca.markets/disclosures

Anthropic (Claude AI)

Enterprise-grade API with data isolation per request. Anonymised spending summaries sent to the API are not used to train models under our API agreement. anthropic.com/privacy

7. Vulnerability Disclosure

If you discover a security vulnerability in Arkonomy, please report it responsibly by emailing security@arkonomy.com. We ask that you do not publicly disclose the issue until we have had a reasonable opportunity to investigate and remediate it.

We will acknowledge all valid reports and work to address confirmed vulnerabilities promptly.

8. Incident Response

In the event of a security incident that affects your personal data, we will notify affected users as required by applicable law and take immediate steps to contain, assess, and remediate the issue.

9. Changes to This Policy

We may update this policy as our security practices evolve. Material changes will be communicated through the app or by email.
